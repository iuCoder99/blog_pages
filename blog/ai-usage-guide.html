为什么需要这个文档？ ​

因为有群友反映不知道如何 ALL IN AI，或者说如何让AI嵌入开发流程，对AI的使用做到心中有数。

主流AI模型介绍 ​

模型工具集成 ​

这些模型集成工具是 Trae、Cursor、Claude Code 等。

商业模式

它们就是工具厂商集中统一向模型提供商购买token，所以你可以看到它们会卖月付套餐。

你也可以自己去第三方平台（阿里云、腾讯云、火山引擎等）或者大模型官网购买token套餐，然后自定义导入。

Vibe Coding 开发流程 ​

主播讲一下现在 Vibe Coding 都是怎么做的。

后端开发 ​

1 接口文档同步

通过 Swagger 注解，AI可以直接查看接口定义。

再通过 IDEA 的 ApiFox插件进行远程同步 ApiFox上面的项目。

这样AI修改你的接口，就可以很简单的实现接口文档同步。

// IDEA插件市场搜索Apifox - API 文档工具 
<span class="comment">// 操作步骤</span>
1. 在 ApiFox 中新建授权秘钥
2. 在插件里连接到你的 ApiFox 账号
3. 在 ApiFox 里新建项目，选择这个项目
4. 点击左边的按钮，进行项目扫描
5. 点击同步按钮，直接就可以进行接口文档同步

2 数据库操作

常见问题：你根据业务自己修改了数据库表结构（通过 DataGrip）。

最淳朴的方式：从 DataGrip 进行表结构导出。

但是AI在写业务的时候，也没办法直接修改数据库，只能给你说，然后你还需要自己改，很麻烦。

💡 MCP 工具解决方案

在 Trae 这种 IDE 里，是可以在自己的环境里面配置模型调用的 MCP 工具的。

MCP（Model Context Protocol）：AI调用工具链来实现和第三方中间件交互。

MCP 工具集成 ​

通过 MCP，AI可以随时读取或者修改数据库。

安全性

可以选择赋予读或者写权限来保证数据操作安全。

优势

相比在终端通过命令行操作，MCP有两个优点：

  1. 速度快
  2. 省 token

Trae MCP 配置步骤 ​

📝 MCP 配置教程

  1. 点击 设置
  2. 找到 MCP 选项
  3. 点击 添加，然后从市场上下载即可
  4. 配置只需要一个 JSON 文件就行
  5. 直接把数据库连接发给AI，AI给你写好JSON代码，你导入即可
// 数据库MCP配置示例 { "mcpServers": { "database": { "command": "npx", "args": ["-y", "@modelcontextprotocol/server-sqlite"], "database": "./my-project.db" } } }

💡 提示：MCP可以设置为项目级，保证在不同项目之间MCP不冲突。

MCP 生态

MCP就是AI模型连接外部工具获取数据、修改数据，所以MCP这么大的市场，都是可以自己摸索着玩的。

项目规则和速览 ​

📋 项目规则
关于一些规范

比如你想让AI按照你的模块规划进行新的业务模块编写，或者按照你的编码风格（例如必须加注释等）。
👁️ 项目速览
关于项目的结构和必要的通用组件说明

方便AI在新的对话中可以快速理解项目，然后着手自己的模块开发。

最佳实践

当AI在这次会话结束时，让它补充有必要的部分到文件中。这样每次新会话都能快速上手项目。

总结 ​

🎯 Vibe Coding 核心要点

✅ 模型选择：根据预算和需求选择合适的AI模型
✅ 工具集成：通过 MCP 让AI连接各种开发工具
✅ 接口同步：使用 ApiFox + IDEA插件实现文档自动同步
✅ 项目规则：建立清晰的项目规范和速览文档
✅ 实践驱动：让AI根据实际业务需求学习和调整

Vibe Coding 就是让AI成为你的开发助手，而不是替代你的思考。
工具只是手段，真正的核心还是你的业务理解和技术判断力。
